## 一、OpenClaw 节点架构

OpenClaw 采用 **Gateway + Node** 架构：

```
┌─────────────────────────────────────────────────────────────┐
│                      Gateway                                │
│  - WebSocket Server (端口 18789)                          │
│  - 消息路由                                                │
│  - 设备配对                                                │
│  - 权限管理                                                │
└───────────────────────┬─────────────────────────────────────┘
                        │
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
   ┌─────────┐    ┌─────────┐    ┌─────────┐
   │ iOS     │    │ Android  │    │ macOS   │
   │ Node    │    │ Node     │    │ Node    │
   └─────────┘    └─────────┘    └─────────┘
```

### 节点角色

| 角色 | 描述 |
|------|------|
| operator | 控制平面客户端（CLI/UI/自动化） |
| node | 能力主机（相机/屏幕/位置/系统命令） |

## 四、分布式组网参考

### 4.1 OpenClaw 连接方式

| 方式 | 协议 | 端口 |
|------|------|------|
| 本地 | WebSocket | 18789 |
| 局域网 | WebSocket + mDNS | 18789 |
| Tailscale | WebSocket + MagicDNS | 18789 |
| SSH 隧道 | WebSocket over SSH | 转发 |

### 4.2 节点发现流程

```
1. 启动 Gateway (ws://0.0.0.0:18789)
2. Gateway 广播服务 (_openclaw-gw._tcp)
3. 节点发现 Gateway
4. 节点发起配对请求
5. Gateway 批准配对
6. 节点连接成功
```

## 五、OpenClaw 远程控制 API

### 5.1 HTTP Tools Invoke

```bash
curl http://gateway:18789/tools/invoke \
  -H 'Authorization: Bearer TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "tool": "sessions_list",
    "args": {}
  }'
```

### 5.2 可用工具

| 工具 | 描述 |
|------|------|
| sessions_list | 列出会话 |
| sessions_send | 发送消息 |
| chat.send | 发送聊天消息 |
| message | 消息操作 |
| gateway | 网关操作 |
| exec | 执行命令 |
| browser | 浏览器控制 |
