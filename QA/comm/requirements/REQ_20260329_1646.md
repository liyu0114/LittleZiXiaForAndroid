# 需求提案

## 时间: 2026-03-29 16:46

## 标题: 超长对话话题管理系统

---

## 需求描述

### 问题背景

用户（Liyu）反馈：在超长时间里，多个话题交织在一个对话框中，会导致：
1. **记忆混乱** — 养生话题和股票话题内容混淆
2. **上下文混乱** — LLM 无法区分当前讨论的是哪个话题
3. **回溯困难** — 想回到10天前的养生话题，找不到入口

### 用户期望

类似市面上 AI 软件的话题功能：
- 用户可以选择不同话题进行对话
- 话题之间内容隔离，不混淆
- 可以随时切换回之前的话题
- 历史话题可搜索、可查看

## 现状分析

### 已有代码

项目已有 `lib/services/conversation/topic_manager.dart`，实现了：
- ✅ 话题创建、切换、归档、删除
- ✅ 按日期分组（今天/昨天/本周/本月/更早）
- ✅ 话题搜索
- ✅ 记忆压缩（消息数超限时自动摘要）

### 缺失部分

| 功能 | 状态 | 说明 |
|------|------|------|
| UI 话题列表界面 | ❌ 无 | 用户无法看到和选择话题 |
| UI 话题切换入口 | ❌ 无 | 侧边栏或抽屉导航 |
| 话题标题自动生成 | ❌ 无 | 需 LLM 根据对话内容生成 |
| 话题间记忆隔离 | ⚠️ 部分 | 需验证是否真正隔离 |
| 话题标签/分类 | ❌ 无 | 养生/股票/工作等分类 |
| 话题智能推荐 | ❌ 无 | 根据输入推荐相关历史话题 |

## 具体需求

### P0 — 核心 UI（必须）

**1. 话题列表侧边栏**

```
┌─────────────┬────────────────────────┐
│ 话题列表    │ 当前对话               │
│ ─────────── │ ──────────────────────  │
│ 🔥 今天     │ 用户: 今天我想谈养生... │
│  └ 养生     │ 助手: 好的...          │
│  └ 股票     │                        │
│ ─────────── │                        │
│ 📅 昨天     │                        │
│  └ 工作汇报 │                        │
│ ─────────── │                        │
│ 📆 本周     │                        │
│  └ 旅行计划 │                        │
│             │                        │
│ [+ 新话题]  │                        │
└─────────────┴────────────────────────┘
```

**2. 话题切换动画**

- 切换话题时清空当前对话显示
- 加载目标话题的历史消息
- 显示过渡动画

**3. 话题标题自动生成**

- 首条消息发送后，调用 LLM 生成话题标题
- 或用户手动编辑

### P1 — 增强功能

**4. 话题标签系统**

```dart
enum TopicCategory {
  health,    // 养生
  finance,   // 理财/股票
  work,      // 工作
  travel,    // 旅行
  learning,  // 学习
  other,     // 其他
}
```

**5. 话题智能推荐**

- 用户输入"养生"时，推荐历史养生话题
- 根据关键词匹配

**6. 话题合并**

- 允许将多个相关话题合并
- 保留完整历史

### P2 — 高级功能

**7. 话题导出**

- 导出为 Markdown / PDF
- 分享话题内容

**8. 话题模板**

- 预设话题模板（周报、日记、学习笔记等）
