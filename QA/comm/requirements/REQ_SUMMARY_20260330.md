### 三、无障碍服务（P1，20h）

**需求来源:** Liyu 指示（2026-03-30 10:30）
**详细文档:** `QA/comm/requirements/REQ_20260330_1030_ACCESSIBILITY.md`

#### 3.1 需求背景

让小紫霞像人一样能够使用手机上已安装的 APP，通过官方合法渠道实现。

#### 3.2 技术方案

**前台服务 (Foreground Service):**
- microphone - 后台持续监听音频
- mediaProjection - 实时捕获屏幕内容
- phoneCall - 管理通话
- shortService - 短时关键任务

**无障碍服务 (Accessibility Service):**
1. 感知屏幕内容 - 获取屏幕上所有 UI 元素
2. 模拟用户操作 - 点击、长按、输入、滑动
3. 监听全局事件 - 应用切换、通知弹出

#### 3.3 开发计划

| Phase | 任务 | 工时 | 内容 |
|-------|------|------|------|
| 1 | 基础框架 | 6h | 创建 AccessibilityService |
| 2 | 屏幕感知 | 4h | 解析 UI 元素树 |
| 3 | 操作执行 | 4h | 点击、输入、滚动 |
| 4 | 应用知识库 | 6h | 微信、支付宝操作脚本 |

**总工时:** 20h

#### 3.4 待确认问题

1. 优先实现哪些应用的操作脚本？（微信、支付宝？）
2. 是否需要屏幕截图能力（mediaProjection）？
3. 权限申请流程如何设计？

**状态:** ⏳ 待 Liyu 确认后开始开发

### 四、记忆系统升级 R1（P1，6h）

**需求来源:** 研究员研究（2026-03-29）
**详细文档:** `QA/comm/QA_20260329_1630_READ_BY_DEV_20260330_1316.md`

#### 4.1 需求背景

对标 OpenClaw 的 MEMORY.md 机制，实现两层记忆架构。

#### 4.2 实现方案

**文件结构：**
```
memory/
├── MEMORY.md          # 长期记忆
├── 2026-03-30.md      # 每日日志
└── 2026-03-29.md      # 昨日日志
```

**加载策略：**
- 启动时加载当日和昨日日志
- 主会话加载 MEMORY.md
- 文件即上下文，持久化能力强

#### 4.3 技术要点

- 使用 `path_provider` 获取应用目录
- 使用 `shared_preferences` 存储轻量数据
- 使用本地文件存储长期记忆

#### 4.4 风险评估

| 风险 | 影响 | 可能性 | 缓解措施 |
|------|------|--------|----------|
| 文件系统性能 | 中 | 中 | 数据压缩 + 延迟加载 |
| 数据丢失 | 高 | 低 | 自动备份 |
| 隐私问题 | 高 | 低 | 本地加密 |

**状态:** ⏳ 待开始

### 五、任务执行框架 R2（P1，9h）

**需求来源:** 研究员研究（2026-03-29）
**详细文档:** `QA/comm/QA_20260329_1630_READ_BY_DEV_20260330_1316.md`

从"咨询师"变为"执行者"，让小紫霞能真正执行任务。

#### 5.2 实现方案

**现状：**
- 代码已存在：`lib/services/agent/task_decomposer.dart`
- 未被实际调用

**改进方案：**
1. 在 AgentOrchestrator 中集成 TaskDecomposer
2. 支持 ExecutionMode 切换（简单/计划模式）
3. 实现步骤执行和进度反馈
4. 执行结果反馈给用户确认

```dart
enum ExecutionMode {
  simple,   // 简单模式：直接执行
  planned,  // 计划模式：分解步骤逐步执行
}

  Future<List<Task>> decompose(String task);
  Future<TaskResult> executeStep(Task task);
  Future<void> reportProgress(int current, int total);
}
```

### 六、权限边界 R3（P2，4h）

**需求来源:** 研究员研究（2026-03-29）
**详细文档:** `QA/comm/QA_20260329_1630_READ_BY_DEV_20260330_1316.md`

#### 6.1 需求背景

让 Agent 有自主能力，但需要明确权限边界。

#### 6.2 实现方案

**权限等级：**
- **L0** - 仅回复（当前状态）
- **L1** - 可执行预定义操作（发消息、查日历）
- **L2** - 可执行敏感操作（需确认）

#### 6.3 技术要点

```dart
enum PermissionLevel {
  level0,  // 仅回复
  level1,  // 预定义操作
  level2,  // 敏感操作（需确认）
}

class PermissionManager {
  bool canExecute(String action, PermissionLevel level);
  Future<bool> requestConfirmation(String action);
}
```
