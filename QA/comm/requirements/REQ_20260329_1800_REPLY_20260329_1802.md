## 技术方案确认

### 1. Server 选择

**建议：用户手动选择**

理由：
- 自动选择需要复杂的选举算法
- 手动选择更简单可靠
- 用户知道哪台设备网络更好

实现：
```dart
enum NodeRole {
  server,   // 服务器节点
  client,   // 客户端节点
  auto,     // 自动（待定）
}
```

### 2. 连接方式

**建议：手动输入 + 二维码扫描**

| 方式 | 优点 | 缺点 |
|------|------|------|
| 手动输入 IP:Port | 简单 | 用户需知道 IP |
| 二维码扫描 | 方便 | 需要摄像头权限 |
| mDNS 自动发现 | 自动 | 仅限局域网 |

**推荐方案：**
1. Server 生成二维码（包含 IP:Port）
2. Client 扫描二维码连接
3. 备用：手动输入

### 3. OpenClaw API

OpenClaw 提供的接口：
- `POST /api/chat` - 发送消息
- `GET /api/status` - 获取状态
- `POST /api/skills/execute` - 执行技能
- `WS /ws` - WebSocket 连接

### 4. 数据同步

**建议：基于 WebSocket 的消息广播**

```dart
class Message {
  final String type;     // chat/skill/status
  final String from;     // 发送者 ID
  final String? to;      // 接收者 ID（null = 广播）
  final dynamic payload; // 消息内容
  final DateTime timestamp;
}
```
