## 需求描述

### 研究结论

经过深入研究 OpenClaw 源码和文档，发现小紫霞与 OpenClaw 的核心差距在于**执行力**：

| 能力 | OpenClaw | 小紫霞 | 差距 |
|------|----------|--------|------|
| Agent Loop | ✅ 完整的思考-行动-观察循环 | ⚠️ 有代码但未完善 | 需增强 |
| 任务分解 | ✅ 自动分解复杂任务 | ✅ 已实现 | OK |
| 工具调用 | ✅ 丰富的内置工具 | ⚠️ 工具较少 | 需扩展 |
| 流式输出 | ✅ 实时流式返回 | ❌ 无 | 需实现 |
| 循环检测 | ✅ 多级阈值+冷却 | ✅ 已实现 | OK |
| 生命周期事件 | ✅ start/end/error | ❌ 无 | 需实现 |
| 任务队列 | ✅ 会话级+全局队列 | ❌ 无 | 需实现 |
| 钩子系统 | ✅ 丰富的 hook 点 | ❌ 无 | 需实现 |

### 核心问题

**小紫霞现在是一个"咨询师"，不是"执行者"。**

- 用户说"帮我查天气"→ 小紫霞能做 ✅
- 用户说"先查北京天气，再查上海天气，最后对比"→ 小紫霞容易漏掉某一步 ❌
- 用户说"帮我写个 Python 脚本并运行它"→ 小紫霞无法真正执行 ❌

### 目标

让小紫霞从"咨询师"变成"执行者"：
1. 能**分解**复杂任务
2. 能**真正执行**每一步
3. 能**反馈进度**给用户
4. 能**处理错误**并重试

## 具体需求

### 1. 完善生命周期管理

参考 OpenClaw 的 `lifecycle` 事件：

```dart
enum AgentLifecyclePhase {
  start,    // 任务开始
  thinking, // 正在思考
  acting,   // 正在执行工具
  observing,// 正在观察结果
  end,      // 任务完成
  error,    // 发生错误
}

class AgentLifecycleEvent {
  final AgentLifecyclePhase phase;
  final String? stepId;
  final String? message;
  final double? progress; // 0.0 - 1.0
}
```

### 3. 添加任务队列

防止并发执行导致的状态混乱：

```dart
class AgentQueue {
  final _sessionQueue = <String, Future<AgentResult>>{};
  
  Future<AgentResult> enqueue(String sessionId, Future<AgentResult> Function() task) async {
    // 等待同一会话的其他任务完成
    await _sessionQueue[sessionId];
    
    final completer = Completer<AgentResult>();
    _sessionQueue[sessionId] = completer.future;
    
    try {
      final result = await task();
      completer.complete(result);
      return result;
    } catch (e) {
      completer.completeError(e);
      rethrow;
    } finally {
      _sessionQueue.remove(sessionId);
    }
  }
}
```

### 4. 增强错误处理和重试

```dart
class RetryHandler {
  final int maxRetries;
  final Duration delay;
  
  Future<T> withRetry<T>(Future<T> Function() action) async {
    int attempts = 0;
    while (true) {
      try {
        return await action();
      } catch (e) {
        attempts++;
        if (attempts >= maxRetries) rethrow;
        await Future.delayed(delay * attempts);
      }
    }
  }
}
```

## 移动端适配

以下功能不适合手机或需要替代方案：

| OpenClaw 功能 | 移动端问题 | 替代方案 |
|--------------|-----------|----------|
| 浏览器自动化 | 资源占用大 | WebView 简化版 |
| 子进程执行 | Android 限制 | Isolate + Platform Channel |
| 文件系统访问 | 权限限制 | scoped_storage + SAF |
| 沙盒执行 | 容器不支持 | Isolate 隔离 |
