## OpenClaw 发现与传输机制

OpenClaw 有两个发现与传输问题：
1. **Operator 远程控制**：macOS 控制其他机器的 Gateway
2. **Node 配对**：iOS/Android 节点发现并配对 Gateway

### 发现方式

| 方式 | 范围 | 说明 |
|------|------|------|
| Bonjour/mDNS | 局域网 | 自动发现，跨网络无效 |
| Tailscale | 跨网络 | MagicDNS 或固定 IP |
| 手动/SSH | 通用 | 端口转发，备用方案 |

### Bonjour 服务类型
- `_openclaw-gw._tcp`

### TXT 记录（示例）
```
role=gateway
lanHost=<hostname>.local
sshPort=22
gatewayPort=18789
tailnetDns=<magicdns>
```

### 客户端传输选择策略

1. 如果有已配对的直接端点且可达 → 使用直接连接
2. 否则，如果 Bonjour 发现 Gateway → 提供"使用此 Gateway"选项
3. 否则，如果配置了 tailnet DNS/IP → 尝试直接连接
4. 否则，回退到 SSH

### 安全说明
- Bonjour/mDNS TXT 记录是**未认证**的，只能作为 UX 提示
- 路由应优先使用**解析后的服务终点**（SRV + A/AAAA）
- TLS 固定不能允许广告的 `gatewayTlsSha256` 覆盖之前存储的固定

## 小紫霞可借鉴的设计

### 方案 A：简化版（类似 OpenClaw）

```
┌─────────────────────────────────────────┐
│           小紫霞 App                    │
├─────────────────────────────────────────┤
│  发现方式：                             │
│  1. 手动输入 IP:Port                    │
│  2. 局域网广播发现（简化 mDNS）         │
│  3. 预设列表快速选择                    │
└─────────────────────────────────────────┘
```

### 方案 B：增强版（分布式 P2P）

```
┌─────────────────────────────────────────┐
│           小紫霞 App                    │
├─────────────────────────────────────────┤
│  Server 模式：                          │
│  - 广播自己的存在                       │
│  - 接受客户端连接                       │
│  - 维护节点列表                         │
├─────────────────────────────────────────┤
│  Client 模式：                          │
│  - 发现可用 Server                      │
│  - 连接到 Server                        │
│  - 收发消息                             │
└─────────────────────────────────────────┘
```

### 方案 C：混合模式

- 手机 A 运行 Server，广播服务
- 手机 B 发现并连接
- OpenClaw 通过 HTTP API 调用接入

|--------|------|--------|
