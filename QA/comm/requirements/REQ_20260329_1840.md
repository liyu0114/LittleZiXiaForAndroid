## 标题: 大模型列表扩展 - 自定义模型支持

---

## 一、问题描述

**现状：**
- 现有大模型列表是预设的，不能穷尽所有好的模型
- 用户无法添加新的模型
- 模型更新需要每次发版

**目标：**
- 让用户可以添加任意自定义模型
- 支持更多大模型提供商
- 灵活配置 API 密钥和端点

---

## 二、OpenClaw 研究成果

### 2.1 OpenClaw 模型配置方式

**方式一：内置 Providers**

OpenClaw 内置支持大量模型提供商：

| Provider | 说明 |
|----------|------|
| openai | OpenAI GPT 系列 |
| anthropic | Anthropic Claude 系列 |
| google | Google Gemini 系列 |
| moonshot | Moonshot Kimi 系列 |
| mistral | Mistral 系列 |
| openrouter | 聚合多个模型 |
| vercel-ai-gateway | Vercel AI Gateway |
| qwen | 阿里 Qwen 系列 |
| ... | 还有更多 |

**方式二：自定义 Provider**

通过 `models.providers` 配置自定义模型：

```json5
{
  models: {
    providers: {
      custom: {
        baseUrl: "https://api.example.com/v1",
        apiKey: "your-key",
        api: "openai-completions"  // 或其他兼容格式
      }
    }
  }
}
```

**方式三：API Key 轮换**

支持多个 API Key 轮换使用：

```bash
OPENAI_API_KEYS=key1,key2,key3
OPENCLAW_LIVE_OPENAI_KEY=override-key
```

## 三、解决方案建议

### 3.1 方案 A：简化版（预设 + 自定义）

```
┌─────────────────────────────────────────┐
│           模型选择界面                   │
├─────────────────────────────────────────┤
│  预设模型列表：                         │
│  - OpenAI 系列                         │
│  - Claude 系列                         │
│  - Gemini 系列                         │
│  - ...                                 │
├─────────────────────────────────────────┤
│  + 添加自定义模型                       │
└─────────────────────────────────────────┘
```

**自定义模型配置项：**

| 字段 | 说明 | 必填 |
|------|------|------|
| 模型名称 | 显示名称 | 是 |
| Base URL | API 端点 | 是 |
| API Key | 密钥 | 是 |
| 模型 ID | 模型标识 | 是 |
| API 类型 | openai/anthropic/custom | 是 |

### 3.2 方案 B：进阶版（Provider 模板）

```
┌─────────────────────────────────────────┐
│           添加模型                      │

│  选择 Provider 模板：                   │
│  ○ OpenAI 兼容                         │
│  ○ Anthropic 兼容                      │
│  ○ 自定义 HTTP                         │
├─────────────────────────────────────────┤
│  填写配置信息...                        │
└─────────────────────────────────────────┘
```

### 3.3 方案 C：完整版（动态加载）

- 支持从配置文件加载模型
- 支持模型市场/社区分享
- 支持模型版本管理

## 五、技术实现建议

### 5.1 数据结构

```dart
class CustomModel {
  String id;
  String name;
  String baseUrl;
  String apiKey;
  String modelId;
  ApiType apiType; // openai, anthropic, custom
  DateTime createdAt;
}

enum ApiType {
  openAI,      // OpenAI 兼容
  anthropic,   // Anthropic 兼容
  custom,      // 自定义 HTTP
}
```

### 5.2 存储方案

| 方案 | 说明 |
|------|------|
| 本地存储 | SharedPreferences（简单） |
| 加密存储 | flutter_secure_storage（安全） |
| 配置文件 | JSON 文件（可导入导出） |

### 5.3 API 调用

```dart
class ModelClient {
  // OpenAI 兼容
  Future<String> callOpenAI(ModelConfig config, String prompt);
  
  // Anthropic 兼容
  Future<String> callAnthropic(ModelConfig config, String prompt);
  
  // 自定义
  Future<String> callCustom(ModelConfig config, String prompt);
}
```
