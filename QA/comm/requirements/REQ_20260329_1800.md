## 标题: 分布式小紫霞网络

## 需求描述

### 背景

小紫霞作为独立 APP，需要能够：
1. 不依赖飞书/OpenClaw，通过配置远程连接
2. 多个小紫霞实例可以互相连接
3. OpenClaw 作为节点加入网络
4. 连接后可以远程控制 OpenClaw

### 核心需求

**1. 独立 APP**
- 小紫霞是独立 APP
- 通过配置连接其他节点
- 不依赖飞书或 OpenClaw

**2. P2P 组网**
- 手机间互相连接
- 一台作为 Server（有公网或能被发现）
- 其他作为 Client 连接

**3. OpenClaw 加入**
- OpenClaw 作为节点加入网络
- 可以通过 HTTP API 调用 OpenClaw

**4. 远程控制 OpenClaw**
- 连上后当 OpenClaw 的遥控器
- 提供控制面板
- 可以发送指令、接收结果

---

## 技术架构

```
[手机A - Server] ←─── WebSocket ───→ [手机B - Client]
       │
       │
       ├─── HTTP API ───→ [OpenClaw]
       │
       └──────── 控制面板 / 远程遥控 ──────────┘
```

---

## 分阶段实施

### P0（基础）
- 两台手机 WebSocket 直连
- Server/Client 角色切换

### P1（组网）
- 多节点组网
- 消息广播/分发

### P2（接入）
- OpenClaw HTTP API 对接
- 控制面板

### P3（高级）
- 远程遥控
- 状态同步
