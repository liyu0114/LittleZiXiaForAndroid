# 通信协议

程序员与质检员的 GitHub 文件通信机制。

## 目录

```
QA/comm/
```

## 文件命名规则

### 发送文件

| 类型 | 格式 | 示例 |
|------|------|------|
| 普通报告 | `{角色}_{YYYYMMDD}_{HHMM}.md` | `DEV_20260329_1530.md` |
| 紧急报告 | `{角色}_{YYYYMMDD}_{HHMM}_URGENT.md` | `DEV_20260329_1530_URGENT.md` |

### 角色代码

- `DEV` — 程序员
- `QA` — 质检员

### 已读标记

收到方读完文件后，将文件重命名：

```
原文件: DEV_20260329_1530.md
已读: DEV_20260329_1530_READ_BY_QA_20260329_1545.md
```

---

## 文件内容模板

```markdown
# 类型: 工作报告 / 质检报告

## 时间: YYYY-MM-DD HH:MM
## 发送方: 程序员 / 质检员
## 接收方: 质检员 / 程序员
## 标题: xxx

## 正文

(内容)

## 待确认事项

- [ ] 事项1
- [ ] 事项2
```

---

## 通信流程

```
1. 发送方创建文件 → git add & commit & push
2. 接收方 git pull → 读取新文件
3. 接收方重命名文件（加 _READ_BY_xxx） → git add & commit & push
4. 发送方看到已读标记 → 删除原文件（可选）
```

---

## 检查频率

- **工作开始时:** 先 `git pull`
- **工作中:** 每 2-4 小时检查一次
- **紧急事项:** 飞书 @ 提醒 + 文件名加 `_URGENT`

---

## 归档规则

- **归档时间:** 每月1日
- **归档操作:** 将上月所有 `_READ_BY_*.md` 文件移动到 `archive/YYYY-MM/`
- **保留期限:** 6个月（之后可手动清理）

### 归档目录结构

```
QA/comm/
├── archive/
│   ├── 2026-03/
│   │   ├── DEV_20260329_1530_READ_BY_QA_20260329_1545.md
│   │   └── QA_20260329_1545_READ_BY_DEV_20260329_1600.md
│   └── 2026-04/
│       └── ...
├── README.md
└── (当前月份的通信文件)
```

---

## 注意事项

1. **先 pull 再工作** — 避免冲突
2. **小文件** — 单个报告控制在 5KB 内
3. **及时标记已读** — 避免重复处理
4. **紧急用飞书** — GitHub 是异步通信，紧急事项飞书 @ 提醒
5. **定期归档** — 每月1日执行归档操作
6. **飞书通知** — 每次操作后在群内通知

---

## 飞书通知格式

每次在 GitHub 上操作后，在飞书群内发送通知：

| 操作 | 格式 | 示例 |
|------|------|------|
| 发送 | `📤 [角色] 发送 文件名` | `📤 [DEV] 发送 DEV_20260329_1530.md` |
| 已读 | `📥 [角色] 已读 文件名` | `📥 [QA] 已读 DEV_20260329_1530.md` |

---
