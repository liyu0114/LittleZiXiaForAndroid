## 一、研究背景

通过研究 OpenClaw 源码和文档，发现其核心优势：

### 1.1 Workspace 机制
- Agent 有独立工作目录
- 包含: AGENTS.md, SOUL.md, MEMORY.md, USER.md 等
- 文件即上下文，持久化能力强

### 1.2 Memory 系统（两层架构）
- `memory/YYYY-MM-DD.md` - 每日日志
- `MEMORY.md` - 长期记忆（仅主会话加载）
- **关键**: 文件是唯一真相来源

### 1.3 Delegate（委托）架构
- Agent 有独立身份
- 代表人类执行任务
- 有自主权限边界

### 1.4 Cron + Heartbeat
- 定时任务执行
- 定期检查机制
- 主动提醒

---

## 二、小紫霞当前问题

| 问题 | 描述 |
|------|------|
| 记忆碎片化 | 无统一 Memory 文件系统 |
| 执行力弱 | 只能回复消息，不能真正执行任务 |
| 无自主权限 | 所有操作需要用户确认 |
| 任务分解弱 | 无 TaskDecomposer 实际使用 |

---

## 三、需求建议

### 3.1 核心需求（R1）- 记忆系统升级

**目标**: 对标 OpenClaw MEMORY.md

**实现方案**:
1. 创建 `memory/MEMORY.md` - 长期记忆
2. 创建 `memory/YYYY-MM-DD.md` - 每日日志
3. 启动时加载当日和昨日日志
4. 主会话加载 MEMORY.md

**优先级**: P1

---

### 3.2 核心需求（R2）- 任务执行框架

**目标**: 从"咨询师"变为"执行者"

**实现方案**:
1. 实现 TaskDecomposer（已发现未集成）
2. 支持简单模式：直接执行
3. 支持计划模式：分解步骤逐步执行
4. 执行结果反馈给用户确认

**优先级**: P1

---

### 3.3 核心需求（R3）- 权限边界

**目标**: 有自主能力的 Agent

**实现方案**:
1. 定义权限等级：
   - L0: 仅回复（当前状态）
   - L1: 可执行预定义操作（发消息、查日历）
   - L2: 可执行敏感操作（需确认）
2. 用户可配置权限等级

**优先级**: P2

---

### 3.4 核心需求（R4）- Heartbeat 增强

**目标**: 主动提醒

**实现方案**:
1. 实现具体检查项：
   - 天气提醒
   - 日程提醒
   - 位置相关提醒
2. 集成系统通知

**优先级**: P2
